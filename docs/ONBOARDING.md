# Onboarding

## Goal
A new employee performs the core queue workflow in under five minutes without a mandatory tutorial.

## First administrator access
1. Confirm restaurant identity.
2. Create or invite team members according to approved membership model.
3. Review public join link/QR.
4. Open a demonstration service or real service.
5. Complete a guided sample: add → call → seat.

## New staff first access
Show a three-step contextual introduction inside the real queue:
1. Add a customer.
2. Use the row actions to call or seat.
3. Check connection and message status separately.

The walkthrough is dismissible and never blocks work.

## Demonstration data
Demo mode/data must be visibly labelled and isolated from real service data. If product/architecture does not support demo tenancy, use a temporary guided sample that is explicitly removed before live use.

## Contextual learning
- Empty queue explains the first action.
- First notification failure explains that queue state is still safe.
- First conflict explains concurrent updates.
- Tooltips are reserved for unfamiliar icons, not core labels.

## Completion criteria
User can identify service state, add a customer, call, seat, mark no-show and recognise a failed WhatsApp message.
