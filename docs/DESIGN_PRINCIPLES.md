# Design Principles

## Interface character
Calm, operational, legible and neutral. The product should feel dependable rather than decorative.

## Hierarchy
1. Current service status.
2. Next operational action.
3. Active queue and entry states.
4. Exceptions requiring attention.
5. Secondary administration and history.

## Layout rules
- Use a persistent application shell for staff.
- Keep primary queue actions in a stable location.
- Prefer one dominant action per context.
- Use progressive disclosure for notes, audit details and exceptional actions.
- Never encode state using colour alone.
- Keep critical labels visible; do not rely only on icons or tooltips.

## Density
- Desktop: compact but scan-friendly.
- Tablet: medium density with 44×44 px minimum touch targets.
- Mobile staff view: reduced columns, action drawer and sticky primary action.
- Customer view: single-column, large status and one primary action.

## Visual decisions deferred
Brand palette, final typeface, illustration style and marketing visual language are deliberately not fixed in this phase.
