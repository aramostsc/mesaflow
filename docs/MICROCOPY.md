# Microcopy

## Voice
Direct, calm, factual and action-oriented. Use European Portuguese in the production locale, with consistent terminology. Avoid technical jargon and blame.

## Canonical action labels
- Add customer
- Call
- Seat
- Mark as no-show
- Reactivate
- Edit details
- Close intake
- Reopen intake (only if approved)
- Close service
- Leave queue
- Try again
- Undo

## Status labels
Waiting, Called, Seated, No-show, Left queue, Cancelled. Notification: Sending, Sent, Delivered, Not delivered.

## Success messages
- **Customer added to the waiting list.**
- **Customer called.**
- **Customer seated.**
- **Entry reactivated.**
- **Intake closed. Existing entries can still be managed.**

## Error messages
- **Enter a valid phone number.**
- **Enter the number of people in the party.**
- **This entry has already changed. We loaded the latest status.**
- **Customer status updated, but the WhatsApp message wasn’t delivered.**
- **You’re offline. Reconnect to continue.**

## Confirmation copy
**Mark this customer as no-show?**  
They will leave the active Called list. You can reactivate the entry if needed.  
Actions: **Mark as no-show** / **Keep called**.

**Close intake?**  
New customers will no longer be able to join. Existing entries remain available.  
Actions: **Close intake** / **Cancel**.

## Forbidden wording
Avoid “Something went wrong”, “Invalid”, “Submit”, “OK” or “Are you sure?” without a specific explanation.
